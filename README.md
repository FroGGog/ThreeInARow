# Match-3 Game

Проект реализует базовую механику игры жанра Match-3 с акцентом на правильную архитектуру, управление памятью и современные возможности C++17.

## Реализованные критерии

1. **Игровое поле и отрисовка**: Корректное отображение, динамическое масштабирование при изменении размера окна без артефактов.
2. **Поиск и активация цепочек**: Нахождение 3+ объектов по горизонтали и вертикали, анимация разрушения (плавное уменьшение радиуса).
3. **Падение и заполнение**: Плавная анимация падения объектов (гравитация), появление новых объектов сверху с анимацией увеличения (Spawning).
4. **Консистентность поля**: Алгоритм генерации и спавна гарантирует отсутствие начальных совпадений (match-3) по вертикали и горизонтали.
5. **Взаимодействие и Бомба**: 
   - Точная проверка попадания клика в окружность.
   - Механика обмена (swap) соседних элементов.
   - **Бомба**: появляется с вероятностью ~2%, имеет анимацию мерцания (изменение alpha-канала), при клике или попадании в матч уничтожает **все** объекты своего цвета на поле.
6. **Архитектура**: Строгое разделение логики и рендера. Состояния объектов управляются через `std::variant` + `std::visit` (без `if-else` по типам состояний).
7. **Управление памятью**: Реализован шаблонный `MemoryPool` с учетом выравнивания (`alignof`), использованием `placement new`, прямым вызовом деструктора и идеальной передачей аргументов (`std::forward`). Освобождение памяти гарантируется через RAII и кастомный `Deleter` для `std::unique_ptr`.

## Требования для сборки

- **CMake** версии 3.16 или выше
- **Компилятор** с поддержкой C++17 (Clang, GCC или MSVC)
- **Qt 5** (компонент Widgets)

## Как собрать и запустить

**Для macOS / Linux:**
```bash
# 1. Создайте папку для сборки
mkdir build && cd build

# 2. Сконфигурируйте проект (CMake автоматически найдет Qt5)
cmake ..

# 3. Соберите проект
cmake --build .

# 4. Запустите исполняемый файл
./Match3 
```

## Как играть
- **Выбор**: Кликните по любому кругу, чтобы выбрать его (вокруг появится белая обводка).
- **Обмен**: Кликните по соседнему кругу (сверху, снизу, слева или справа), чтобы поменять их местами.
- **Совпадение**: Если после обмена образуется линия из 3 и более кругов одного цвета, они будут уничтожены.
- **Гравитация**: Круги выше упадут вниз, а сверху появятся новые (без создания новых готовых комбинаций).
- **Бомба**:
Иногда при спавне появляется круг с мерцающей белой обводкой.
Чтобы активировать её, просто кликните по ней. Она мгновенно уничтожит все круги своего цвета на поле, вызвав цепную реакцию падения.
Бомба также срабатывает автоматически, если попадает в обычный матч из 3+ элементов.

## Архитектурные особенности
- **State Machine**: Класс CircleObject содержит std::variant<IdleState, FallingState, DestroyingState, SpawningState>. Методы update и render делегируют работу текущему состоянию через std::visit.
- **Memory Pool**: Выделение памяти происходит один раз при инициализации. При создании объекта используется placement new, а при удалении unique_ptr автоматически вызывает кастомный PoolDeleter, который обнуляет флаг занятости ячейки в пуле, обеспечивая полное отсутствие утечек памяти.
- **Отсутствие магических чисел**: Цвета вынесены в enum ObjectColor, размеры сетки заданы константами.
